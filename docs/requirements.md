# 需求文档：PDF to Markdown 转换工具

## 1. 项目概述

本工具将上传的 PDF 文件转换为结构化的 Markdown 文件。通过 LangGraph React Agent 编排整个转换流程，利用 LLM 视觉能力对 PDF 每一页进行深度理解和内容提取。

## 2. 功能需求

### 2.1 核心功能

| 编号 | 功能 | 优先级 |
|------|------|--------|
| F-01 | 将 PDF 各页转换为 JPEG 图像 | P0 |
| F-02 | 从图像中提取文本内容并保留格式结构 | P0 |
| F-03 | 识别并提取表格，输出为 Markdown 表格格式 | P0 |
| F-04 | 识别图片/图表，生成文字描述插入上下文 | P0 |
| F-05 | 将各页结果按顺序组装为单一 Markdown 文件 | P0 |
| F-06 | 命令行入口支持指定输入/输出路径 | P1 |

### 2.2 工具（Tools）定义

#### Tool 1: `pdf_to_images`
- **输入**: PDF 文件路径、输出目录、分辨率（DPI，默认 150）
- **输出**: JPEG 文件路径列表（按页码排序）
- **行为**: 每页渲染为独立 JPEG 文件，文件名格式 `page_{n:03d}.jpg`

#### Tool 2: `analyze_image_page`
- **输入**: JPEG 文件路径、页码
- **输出**: 结构化 JSON 字符串，包含有序的内容块列表
- **行为**:
  - 调用支持视觉的 LLM（如 GPT-4o）
  - 按页面内容的阅读顺序，输出以下类型的内容块：
    - `text`: 提取的文字段落
    - `table`: 转换后的 Markdown 表格
    - `image_description`: 图片/图表/公式的文字描述，格式为 `> 📷 [描述内容]`
  - 保留标题层级（`#`、`##`、`###`）

#### Tool 3: `assemble_markdown`
- **输入**: 各页分析结果（JSON 数组字符串）、输出文件路径
- **输出**: 最终 Markdown 文件路径
- **行为**:
  - 按页码顺序拼接内容
  - 各页之间添加分页分隔符（`---`）
  - 写入输出文件

### 2.3 Agent 行为

- Agent 接收 PDF 路径和期望的输出路径
- 自主决定工具调用顺序和参数
- 对每页独立调用 `analyze_image_page`，支持局部重试
- 所有页分析完成后调用 `assemble_markdown` 输出结果

## 3. 非功能需求

### 3.1 性能
- 单页分析超时：60 秒
- 支持最大 200 页的 PDF

### 3.2 可靠性
- 单页分析失败时，记录错误并继续处理其余页
- 失败页在 Markdown 中以 `> ⚠️ 第 N 页解析失败` 占位

### 3.3 配置
- LLM 模型名称通过环境变量 `LLM_MODEL` 配置（默认 `gpt-4o`）
- PDF 渲染 DPI 通过环境变量 `PDF_DPI` 配置（默认 `150`）
- 临时图像目录通过环境变量 `TEMP_DIR` 配置（默认 `./tmp`）

## 4. 约束

- 运行环境：Python 3.11+
- LLM：需支持视觉输入（Vision）的模型，通过 OpenAI-compatible API 访问
- 依赖的外部服务：OpenAI API（或兼容接口）
