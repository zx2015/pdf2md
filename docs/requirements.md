# 需求文档：PDF to Markdown 转换工具

## 1. 项目概述

本工具将上传的 PDF 文件转换为结构化的 Markdown 文件。通过 LangGraph React Agent 编排转换流程，每页独立 Agent 处理，利用 LLM 视觉能力深度理解 PDF 内容。提供 Web UI 和命令行两种使用方式。

## 2. 功能需求

### 2.1 核心转换功能

| 编号 | 功能 | 优先级 |
|------|------|--------|
| F-01 | 将 PDF 各页转换为 JPEG 图像 | P0 |
| F-02 | 从图像提取文本内容，保留标题层级（#/##/###）、加粗、列表 | P0 |
| F-03 | 识别并提取表格，输出标准 Markdown 表格格式 | P0 |
| F-04 | 识别图片/图表，生成文字描述（`> 📷 `开头），图表转 Mermaid 代码块 | P0 |
| F-05 | 识别数学公式，输出 `$$...$$` LaTeX 格式 | P0 |
| F-06 | 处理双栏布局：先读完左栏，再读右栏，输出单栏连续文本 | P1 |
| F-07 | 跨页拼接：读取输出文件末尾 15 行作为上下文，处理跨页段落/表格/列表 | P0 |
| F-08 | 命令行入口支持指定输入/输出路径 | P1 |

### 2.2 Web 界面功能

| F-16 | 实时进度条（按页推进）| P1 |

### 2.3 可靠性功能

| 编号 | 功能 | 优先级 |
|------|------|--------|
| F-20 | LLM 调用失败时自动重试（网络超时、连接中断、Rate Limit）| P0 |
| F-21 | 单页处理失败时最多重试 3 次，全部失败后记录断点，任务暂停 | P0 |
| F-22 | 断点续传：失败任务记录中断页码，可从该页一键继续 | P0 |
| F-23 | 断点续传时跳过已完成页面的 PDF 重新渲染步骤 | P1 |

### 2.4 工具（Tools）定义

#### Tool: `pdf_to_images`
- **输入**: `pdf_path`（PDF 路径）, `output_dir`（输出目录）, `dpi`（渲染分辨率，默认 150）
- **输出**: JSON 字符串，JPEG 文件路径列表（按页码升序）
- **行为**: 每页渲染为独立 JPEG，文件名格式 `page_{n:03d}.jpg`

#### Tool: `describe_image`
- **输入**: `image_path`（JPEG 路径）, `prompt`（由 Agent 构建的识别指令）
- **输出**: LLM 直接响应的 Markdown 文本
- **行为**:
  - Agent 自行构建 prompt，包含文档背景、上下文、提取要求
  - 包含 tenacity 重试逻辑（网络错误/超时/Rate Limit）
  - 失败后返回 `"⚠️ 错误：..."` 字符串（不抛出异常）

#### Tool: `read_file_lines`
- **输入**: `path`（文件路径）, `start_line`（开始行，支持负数索引）, `end_line`（可选）
- **输出**: 指定范围的文本内容

#### Tool: `write_file_lines`
- **输入**: `path`（文件路径）, `content`（内容）, `mode`（`"append"` 或 `"overwrite"`）
- **输出**: 写入结果描述

### 2.5 Agent 行为

- **每页独立 Agent 实例**，无跨页共享上下文（避免 context window 溢出）
- Agent 接收：当前页图片路径、上一页图片路径（可选）、输出文件路径、页码/总页数
- 步骤：读上文末尾 → 识别当前页 → 自检内容质量 → 写入文件
- 每页 `recursion_limit=20`（足够处理正常页面的所有步骤）
- 单页失败：外层循环重试最多 3 次，后抛 `PageProcessingError`

## 3. 非功能需求

### 3.1 性能
- 单次 LLM 调用超时：120 秒
- 串行逐页处理，不并发（确保拼接顺序正确）
- 支持长文档（实测 100+ 页）

### 3.2 可靠性
- 网络抖动、Rate Limit 自动重试（指数退避）
- 单页失败最多重试 3 次，记录断点后可续传
- SSE 日志持久化到 `logs.jsonl`，断线重连后完整回放

### 3.3 安全性
- API Key 通过 `.env` 环境变量注入，不提交到代码库
- 任务目录隔离：每个任务 UUID 目录，文件互不干扰

### 3.4 配置

通过 `.env` 环境变量配置：

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `OPENAI_API_KEY` | API Key（必填）| — |
| `OPENAI_BASE_URL` | 自定义 API 端点 | OpenAI 官方 |
| `LLM_MODEL` | 视觉模型名称 | `gpt-4o` |
| `PDF_DPI` | PDF 渲染分辨率 | `150` |
| `TASKS_DIR` | 任务存储根目录 | `./tasks` |
| `MAX_CONCURRENT_TASKS` | 最大并发任务数 | `3` |
| `PAGE_TIMEOUT` | 单次 LLM 调用超时（秒）| `120` |
| `RETRY_ATTEMPTS` | LLM 重试总次数（含首次）| `4` |
| `RETRY_WAIT_MIN` | 首次重试等待秒数 | `2` |
| `RETRY_WAIT_MAX` | 最大退避等待秒数 | `60` |
| `RATE_LIMIT_WAIT` | 429 时额外等待秒数 | `15` |

## 4. 约束

- 运行环境：Python 3.11+
- LLM：需支持视觉输入（Vision）的模型，通过 OpenAI-compatible API 访问
- 外部依赖：OpenAI API 或兼容接口（如 LiteLLM、阿里云 DashScope 等）
